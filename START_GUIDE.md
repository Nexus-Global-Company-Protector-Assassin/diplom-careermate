# 🚀 Инструкция: Как запускать и перезапускать проект CareerMate

В этом проекте есть **4 главных компонента**, которые зависят друг от друга. Порядок запуска очень важен!

1. **Docker (База Данных)** — основа всего. Без неё бэкенд вылетит с ошибкой.
2. **Бэкенд (API)** — связующее звено.
3. **AI Агент** — парсинг резюме и диалоги с ИИ.
4. **Фронтенд (Сайт)** — то, что работает в браузере.

---

## 🟢 Повседневный запуск проекта (Быстрый старт)

Если вы только включили компьютер, делайте всё строго по порядку:

**Шаг 1. Включите Docker**
- Запустите приложение **Docker Desktop** из меню «Пуск» в Windows.
- Дождитесь, пока в углу не появится зеленая иконка кита с надписью `Engine running`.

**Шаг 2. Поднимите Базу Данных**
Откройте новый терминал (PowerShell или Cmd) в корне проекта (`diplom-careermate`) и введите:
```bash
npm run docker:dev
```
*Это запустит PostgreSQL, Redis и остальные контейнеры.*

**Шаг 3. Запустите Бэкенд и Фронтенд**
В том же терминале введите команду для старта сайта и API:
```bash
npm run dev
```
*Сайт будет доступен на `http://localhost:3000`, а бэкенд на `3001` порту.*

**Шаг 4. Запустите AI Агента (Отдельно)**
Откройте **новый (второй)** терминал в VS Code, перейдите в папку агента и запустите его:
```bash
cd agent
npm run dev
```
*Агент запустится на порту `3002`. Теперь парсинг резюме работает!*

> 🎉 **Всё! Проект полностью запущен и готов к работе.**

## 🛠 Управление базой данных (Prisma)

Если вам нужно зайти в саму базу и посмотреть таблички (Prisma Studio):

1. Откройте **третий** терминал.
2. Обязательно перейдите в папку бэкенда!
```bash
cd backend
npx prisma studio
```
Студия откроется по ссылке: `http://localhost:5555`.

## 🔴 Как всё правильно выключить (Перезапуск)

Иногда сайт или база начинают "глючить", например, после обновления `.env` или когда завис порт. Делайте **полный перезапуск** по этой схеме:

**1. Выключите процессы в терминалах**
- Нажмите `Ctrl + C` в терминале с сайтом (`npm run dev`).
- Нажмите `Ctrl + C` в терминале с Агентом.
- Нажмите `Ctrl + C` в терминале с Prisma Studio.

**2. Убейте и перезапустите Базу Данных в Docker**
Выполните команду остановки контейнеров:
```bash
npm run docker:down
```
После того как контейнеры удалятся, поднимите их заново на чистую:
```bash
npm run docker:up
```

**3. Запустите всё по шагам Быстрого старта**
(Смотри раздел "Быстрый старт" выше: `npm run dev` для сайта и `cd agent && npm run dev` для ИИ).

## 🚨 Частые Ошибки и Как их лечить

### Ошибка: `Can't reach database server at 127.0.0.1:5433`
**Причина:** Бэкенд загрузился, но не нашел базу данных. Либо выключен Docker Desktop, либо контейнеры не подняты.
**Решение:** Откройте Docker Desktop. Убедитесь, что он работает. Затем выполните `npm run docker:dev`. После этого обязательно нажмите `Ctrl + C` в терминале `npm run dev` и запустите его заново, чтобы бэкенд переподключился.

### Ошибка: `Не удалось распознать документ AI Агентом. Ошибка 500.`
**Причина:** Вы обновляли `.env` ключи нейросети, но не перезагрузили Агента.
**Решение:** В терминале агента нажмите `Ctrl + C` и снова введите `npm run dev`. Процесс Node.js должен загрузить новые ключи при старте.

### Ошибка при запуске Prisma Studio: `There is a conflict between env vars`
**Причина:** Вы запускаете команду из корня проекта, где лежат настройки фронтенда, и они конфликтуют с бэкендом.
**Решение:** Prisma Studio нужно запускать СТРОГО из папки бэкенда: `cd backend` -> `npx prisma studio`.

### Ошибка: `Port 3000 / 3001 is already in use`
**Причина:** Вы закрыли VS Code, не остановив сервер, и процесс завис в памяти Windows.
**Решение:** Откройте PowerShell и "убейте" все node-процессы:
```powershell
taskkill /F /IM node.exe
```
Затем запускайте проект заново.

### Ошибка в Prisma Studio: `The table public.X does not exist in the current database`
**Причина:** База данных либо абсолютно новая, либо контейнер был удален вместе со структурой (например, при первом запуске или полном сбросе).
**Решение:** В терминале в корне проекта (где лежит файл `package.json`) выполните следующие команды:
```bash
npm run db:migrate
npm run db:seed
```
Это накатит схему базы данных (создаст необходимые таблицы) и заполнит ее стартовыми тестовыми данными. После этого просто обновите страницу Prisma Studio.

### Ошибка: `Internal Server Error` при парсинге вакансий с HH.ru
**Причина:** HH.ru блокирует серверные запросы (DDoS Guard).
**Решение:** 
1. **Без токена:** Код автоматически переключится на реалистичные **Mock-данные** (Yandex, Сбер и т.д.), чтобы вы могли тестировать интерфейс.
2. **С реальными данными:** Зарегистрируйте приложение на [dev.hh.ru](https://dev.hh.ru), получите `access_token` и вставьте его в файл `backend/.env` в поле `HH_ACCESS_TOKEN=`.
3. После добавления токена не забудьте перезагрузить бэкенд.
