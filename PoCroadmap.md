# 🧪 PoC Roadmap (TRL3) — CareerMate

Этот PoC нужен, чтобы:
- показать **работающую технологическую идею** (TRL3),
- доказать **понимание AI + архитектуры**,
- иметь **что показать** в заявке / интервью Яндекса,
- не умереть командой из 4 человек.

---

## 🎯 Цель PoC (зафиксировано)

**PoC ≠ MVP ≠ продукт**

PoC = доказательство, что система способна провести пользователя по одному ключевому сценарию,  
где AI **не просто генерирует текст**, а **выполняет действия**.

---

## 🧩 Единственный сценарий PoC (killer flow)

> **Профиль → AI-анализ → подбор вакансий → адаптированное резюме → сохранённый результат**

❌ НЕ делаем:
- интервью
- карьерные треки
- auto-apply
- подписки
- сложный UI
- интеграции с job boards

---

## 🧱 Архитектура PoC

Frontend → Backend API → Agent Service (MCP)
                ↘
                Database


- **Frontend** — показать ввод и результат
- **Backend** — данные, бизнес-логика, API
- **Agent** — MCP, AI, оркестрация, генерация

## 🟩 Неделя 1: Skeleton Backend + Agent

### Backend
**Задачи:**
- NestJS backend
- Сущности:
  - User
  - Profile (skills, experience, target role)
  - AnalysisResult
  - Resume
- Эндпоинты:
  - `POST /profiles`
  - `POST /poc/run`

### Agent
**Задачи:**
- Отдельный NestJS сервис (`agent`)
- LLM Gateway (generateText / generateJson)
- MCP Tool:
  - `analyze_profile`
- Строгая JSON-схема ответа

### Артефакт недели
- Backend и Agent поднимаются
- Backend может вызвать Agent
- Возвращается валидный JSON

📌 **TRL3 достигнут (базовый PoC)**

---

## 🟩 Неделя 2: Matching + Resume Generation

### Agent (ключевая часть)
**Добавить MCP tools:**
1. `analyze_profile`
2. `match_vacancies`
3. `generate_resume`

⚠️ Источник вакансий:
- захардкоженный список
- CSV
- 20–30 ручных примеров  
❌ без парсинга и API

**Требования:**
- строгие JSON-схемы
- repair-логика при невалидных ответах
- логирование шагов агента (`runId`)

### Backend
- Сохранение:
  - анализа
  - топ-вакансий
  - резюме (text/html)
- Эндпоинт:
  - `POST /poc/run → { analysis, vacancies, resume }`

### Артефакт недели
- Один запрос → полный результат
- Демонстрация через Postman / curl

📌 **Сильный TRL3 PoC**

## 🟩 Неделя 4: Подготовка к подаче в Яндекс

### Задачи
- Записать демо-видео (2–3 минуты)
- Подготовить метрики PoC
- Сформулировать:
  - что доказали
  - какие риски
  - план на 6–8 недель

### Метрики PoC
**Технические:**
- ≥90% успешных прогонов
- p95 < 10 секунд
- backend не падает при ошибках агента

**Качественные:**
- ≥80% вакансий релевантны (ручная оценка)
- 0 выдуманных фактов в резюме
